Users

Contracts are made with a client only. If you want to make a contract with more, they will have to make a user together.

Even so, you are able to mention their names either within firstname / lastname of the user, or in contract details.

Users have:

  • Username
  • Password
  • Firstname
  • Lastname
  • Email
  • Type ( client, employee or admin )

( and an id )

Restrictions are not made concerning how many users ( clients / employees / admins ) there might be.

User types

User types are: client, employee, admin.

Clients: are able to view "client views", where they are shown their contracts with their part progress and details.

Employees: are able to view almost as much as an admin, but they are only allowed to edit part progress / part tasks of contracts that were assigned to them by an admin.
Employees are not able to make contracts, parts for them or give themselves tasks. However, they are able to complete parts or finalise contracts.

Admins: are able to make contracts, parts for them, give tasks to themselves and employees as well as edit anything.

Parts

Parts are what a contract is made of.

Parts have:

  • Part name
  • Client
  • Progress

( and an id )

In order to make a part, you have to also add it to a contract. So, you have to have a contract to make a part for it.

Parts are completed when they have reached 100% in progress.

Part types

Part types are: completed or not completed.

Completed parts: are parts thath are 100% completed.

Users are not shown tasks for Completed Parts!

Contracts

Contracts are made with a client only. If you want to make a contract with more, they will have to make a user together.

Even so, you are able to mention their names either within firstname / lastname of the user, or in contract details.

Contracts have:

  • Contract name
  • Client
  • Details
  • Contract date
  • Deadline date

( and an id )

Contract types

Contract types are: obsolete, finalised, finalised and obsolete, interesting, final month and deleted.

Obsolete Contracts: are contracts that have passed their deadline date. Decide if you delete / edit deadline

Finalised ( or Done ) Contracts: are contracts that have all parts completed ( meaning all parts are 100% finalised ). Delete

Finalised ( or Done ) and Obsolete Contracts: are contracts that are both done and obsolete, meaning that you should delete them. Delete / edit deadline

Interesting Contracts: are contracts that are not obsolete or finalised ( done ).

Final Month Contracts: are contracts that have their deadline month.

Deleted Contracts: are contracts that were deleted, of course. When an admin deletes a contract, all of its details ( parts, part tasks, deadline, client name ) are saved in a format that is not editable. Also, admins are not able to delete a deleted contract.

Users are not shown tasks for Finalised Contracts!

Tasks / Part tasks

( Tasks are given on Contract Parts. So, you have to have a Contract, then Parts for it then Tasks for the parts. )

( Part Tasks are given on Tasks. So, you have to have a tasks for all contract parts, then add part tasks to those. )

Even so, it is not complcated: Give tasks on Contract Parts.

Tasks have:

  • User ( employee or admin )
  • Part ( that you have made within a contract )

( and an id )

Part tasks have:

  • Details ( a line with the actual task )
  • Task ( telling us what contract Part and User the task is for )

You cannot give the same task to the same user on the same contract part twice.

Part task types

Part task types are: given and completed.

Given part tasks: are part tasks that were not yet completed

Completed part tasks: are part tasks that were said to be completed ( either by the task user or any admin ).

Completing tasks does not complete Parts! ( Part progress and Part tasks are separated completly ).
